Este workflow deve ser executado sempre que uma tarefa significativa for concluída para garantir que a inteligência do projeto evolua.

1.  **Consultar Lições:** Leia `lessons_learnings.md` para verificar se a solução atual gerou um novo padrão ou evitou um erro antigo.
2.  **Atualizar Diretrizes:** Se um novo componente UI ou padrão de backend foi criado, adicione-o a `directives.md`.
3.  **Refinar PRD:** Marque como concluído as etapas no Roadmap do `PRD_Portal_Eventos_Patos_de_Minas.md`.
4.  **Confirmar RLS:** Verifique se todas as tabelas afetadas possuem políticas de Row Level Security ativas.

5.  **Commit Governance:** Sugira um commit específico para as atualizações de governança se houverem alterações nestes arquivos.
